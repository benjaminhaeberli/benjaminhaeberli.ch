# GitMoji Commit

Create a commit with GitMoji convention.

## Input
$ARGUMENTS

## Instructions

1. Analyze the staged changes with `git diff --staged`

2. Determine the appropriate GitMoji:
   - :sparkles: New feature
   - :bug: Bug fix
   - :art: Code style/format
   - :recycle: Refactoring
   - :arrow_up: Upgrade dependency
   - :memo: Documentation
   - :lipstick: UI/Style changes

3. Create a concise commit message:
   - Format: `:emoji: description`
   - Use imperative mood
   - Keep under 72 characters

4. Execute the commit

5. Show the commit result
